# ML-Salekhard

## Цель проекта
В условиях деградации многолетней мерзлоты критически важно прогнозировать температурное состояние грунтового основания зданий. Численные расчеты и детерминированные физические модели (сложные, требующие много параметров) могут быть дополнены моделями, способными выявлять сложные нелинейные зависимости в данных мониторинга. Задача проекта — создать прототип такой модели для конкретных объектов (жилых зданий) в Салехарде.

Основная цель проекта: разработать модель машинного обучения для прогнозирования температуры грунта, которая включает как сбор и предобработку данных, так и анализ и разработку критериев оценки и возможных сценариев развития теплофизических процессов.

Результаты проекта будут использоваться в учебном процессе для спецкурса "Компьютерное моделирование и вычислительные методы в задачах теплофизики и их приложения в Арктике".

## Результат (продукт)
Разработать модель машинного обучения для прогнозирования температуры грунта на любой глубине в термометрической скважине на основе температуры воздуха в подполье в конкретный выбранный месяц или иной период года.

Разработка продукта предполагает следующие этапы.

1. Сбор и предобработка данных: получить доступ к серверу мониторинга, выгрузить временные ряды температур со всех датчиков (глубинных + подполье) за 3-5 лет. Обработать пропуски, аномалии, синхронизировать данные.

2. Инжиниринг признаков (Feature Engineering): создать набор признаков, наиболее информативных для прогноза (месяц, глубина, температура подполья, скользящие статистики, лаговые значения).

3. Исследовательский анализ данных (EDA): визуализировать тепловые волны в грунте, определить амплитуду затухания, временные лаги, сезонные и годовые тренды.

4. Разработка и обучение моделей: выбрать и обучить несколько типов моделей (от классических регрессий до нейросетей), способных работать с пространственно-временными данными.

5. Валидация и оценка модели: протестировать модель на "невидимых" данных (например, на 2026 году), оценить качество прогноза ключевыми метриками.

6. Создание инструмента прогнозирования: разработать прототип веб-интерфейса или скрипта, позволяющего по введенной дате/месяцу и температуре в подполье получать прогнозный температурный профиль по глубине.

7. Анализ результатов и выводы: сформулировать выводы об устойчивости мерзлоты под конкретными зданиями, дать рекомендации по мониторингу и использованию модели.

## Требование к результату (продукту)
Ожидаемые результаты и применение

1. Набор данных: Очищенный, аннотированный датасет температурных профилей, пригодный для будущих исследований.

2. Обученная модель: Файл с лучшей моделью (.pkl, .h5), достигающая RMSE < 0.5°C на тестовом периоде.

3. Визуализации: Графики сезонной динамики, тепловые карты (глубина-время), важности признаков, сравнения прогноза и реальности.

4. Прототип веб-приложения (опционально, но желательно): Простой интерфейс на Streamlit/Gradio, где пользователь выбирает месяц, вводит температуру в подполье и получает прогнозный разрез.

5. Отчет/статья: Структурированный отчет с описанием методики, кода, выводов. Основа для научной публикации или доклада на конференции.
